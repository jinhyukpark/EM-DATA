# Permission Management — UI 개선 스펙

## 모달 기본 구조

```
Role Name 입력 필드
→ 권한 구분 라디오 버튼
→ 선택에 따른 하위 항목
→ 하단: Cancel / Save Role 버튼
```

---

## Role Name

- 텍스트 입력 필드
- placeholder: `e.g. Content Manager`

---

## 권한 구분 (라디오 버튼)

두 옵션 중 하나만 선택 가능. 선택에 따라 하위 영역 전환.

| 옵션 | 배지 | 설명 |
|------|------|------|
| 관리자 권한 | `슈퍼 관리자` (보라색) | 시스템·그룹 관리 권한 설정 |
| 서비스 이용 권한 | 없음 | 메뉴별 View/Create/Edit/Delete/Export 설정 |

---

## [관리자 권한 선택 시]

### 섹션 1 — 시스템 관리 `Admin only`

각 항목은 토글 스위치(on/off)로 제어.

| 항목 | 설명 | 기본값 |
|------|------|:------:|
| Permission Management | 서비스 이용 권한 role 설정 (관리자 권한 제외) | ON |
| User Management | 구성원 초대·제거·역할 변경 | ON |
| Email Notifications | 시스템 알림 설정 관리 | OFF |

### 섹션 2 — 그룹 관리 `Group Admin`

각 항목은 토글 스위치(on/off)로 제어.

| 항목 | 설명 | 기본값 |
|------|------|:------:|
| QA Report 그룹 관리 | inspector 배정, 일정·검사 항목 관리 | OFF |
| Internal Data 그룹 관리 | 데이터 항목 전체 관리 | OFF |
| Server Management 그룹 관리 | 서버 항목 전체 관리 | OFF |

---

## [서비스 이용 권한 선택 시]

체크박스 테이블. 컬럼: Module / View / Create / Edit / Delete / Export / All

- `All` 컬럼의 Toggle 클릭 시 해당 행 전체 체크박스 일괄 on/off
- 그룹 헤더 행으로 섹션 구분 (배경색으로 시각적 분리)

## 모달 하단 버튼

- `Cancel` — 변경사항 저장 없이 모달 닫기
- `Save Role` — 변경사항 저장 (primary 버튼, 우측 정렬)
